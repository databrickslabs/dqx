## Global Constraints
- i18n: every user-facing string in ALL 4 locales (`ui/lib/i18n/locales/{en,es,it,pt-BR}.json`), real translations; en is source of truth.
- Backend model changes → `make app-regen-api` (never hand-edit `ui/lib/api.ts`).
- Never hand-edit generated `routeTree.gen.ts` (regenerates).
- GPG-sign every commit; message ends with `Co-authored-by: Isaac`. Never `--no-gpg-sign`.
- `app/uv.lock` must stay public-URL only — `git checkout -- app/uv.lock` if a local build dirties it; check `git diff app/uv.lock` before each commit.
- No `# type: ignore` / `# noqa` / lint-disable to silence issues (per AGENTS.md).
- Keep suites green: `make app-check` (tsc + basedpyright), `make app-test` (backend pytest), bun tests.
- Verification: deploy per theme to `fe-sandbox-dq-demo`/`dev` and live-verify; if a deploy is problematic (e.g. permission-related), carry on. `app/target.dev.yml` (untracked) must exist — copy from a sibling worktree.

## Execution model
- **Subagent-driven development**: fresh implementer subagent per task → reviewer subagent → fix loop → commit → ledger append. Model: **fable** for implementers if no spend-limit; fall back to **opus 4.8** on spend-limit error (probe fable once at the start).
- **Parallelism**: dispatch implementer subagents concurrently ONLY when their file sets are disjoint. Shared worktree ⇒ concurrent edits to the same file clobber. Use per-agent worktrees (isolation:"worktree") for concurrent streams; orchestrator merges to `ui-tweaks` and resolves locale-JSON conflicts.
- **Locale JSON is the cross-cutting bottleneck** — serialize: at most one concurrent agent edits the locale files per wave, or orchestrator applies locale keys between merges.
- **Ledger**: `.superpowers/sdd/progress.md` (gitignored) — authoritative resume record. Append after every task.
- Order: **easy layout/copy first**, then medium, then the big pieces (13 drafts redesign, 15 runs-history rebuild, 27 settings redesign, 89 AI generator, 21 cross-schedule).
