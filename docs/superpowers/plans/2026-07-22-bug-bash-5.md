# Bug-bash-5 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix a batch of DQX Studio roles/permissions bugs, a profiler→apply-rules UX bug, three run-status bugs, and two small UI changes.

**Architecture:** Roles are a hard ceiling (`require_role` on every route); object grants only refine *within* a role. This plan (a) removes the `RUNNER` role and rebalances the permission matrix, (b) reworks object permissions to store real default grant rows + adds an `EXECUTE` privilege, (c) makes profiler-apply push into the Apply Rules tab's existing unsaved `stagedRows` selection, (d) fixes run-status polling/propagation, (e) does two label/tooltip changes.

**Tech Stack:** FastAPI + Pydantic backend, React 19 + TanStack Query + shadcn/ui frontend, orval-generated `lib/api.ts`, Delta + Lakebase Postgres storage, react-i18next (en/pt-BR/it/es).
