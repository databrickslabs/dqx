## Conventions that bit us / must respect
- Four locale files must stay key-synced: `en/pt-BR/it/es.json` under
  `ui/lib/i18n/locales/`. Pluralize with `_one`/`_other` + `{{count}}`.
- Rule mode internal ids MUST NOT change: `dqx_native` (Basic Rules),
  `lowcode` (Condition Builder / Custom Checks), `sql`.
- Operator STRING is both the persisted key and the display label.
- After backend model changes: `make app-regen-api` (done for the `rule_id[]`
  export param). Axios serializes arrays as repeated params (`{indexes:null}`),
  which matches FastAPI `list[str]`.
- **Low-code operators live in TWO files that must stay in lock-step:**
  `ui/lib/lowcodeOperators.ts` + `ui/lib/lowcodeCompile.ts` (frontend, uppercase
  families) and `backend/lowcode_compile.py` (AI generator, lowercase families).
  Adding an operator to only one side makes it silently unusable (frontend: no
  SQL; AI: not proposed / rejected at validation).
