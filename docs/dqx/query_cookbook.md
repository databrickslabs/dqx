# DQX Query Cookbook

> A practical guide to common data quality patterns using DQX.

## Overview

This cookbook provides ready-to-use examples for common data quality validation scenarios. Each recipe includes the check definition, expected behavior, and usage tips.

## Planned Recipes

- **Null & Empty Checks** — validate required fields across tables
- **Range & Threshold Checks** — ensure numeric columns stay within bounds  
- **Regex Pattern Matching** — validate formats (emails, phone numbers, IDs)
- **Cross-Column Consistency** — checks that depend on multiple columns
- **Referential Integrity** — foreign key validation patterns
- **Freshness Checks** — ensure data is recent enough
- **Anomaly Detection Integration** — combining profiler output with custom checks
