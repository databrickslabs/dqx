# Monitor your tables

This section is for stewards who want to put real, ongoing data-quality checks on
a table. It's where the reusable rules in your [Rules Registry](/docs/studio/start-here/key-concepts)
stop being definitions and start doing work.

When you apply rules to a Unity Catalog table, that table becomes a **monitored
table** — the single source of truth for that table's data quality. Everything
about how healthy the table is, which checks run on it, and how it's trended over
time lives in one place.

## Open a table to monitor

1. Go to **Tables**.
2. Select **Monitor a table**.
3. Browse Unity Catalog: pick a **catalog**, then a **schema**, then a **table**.
   You only see catalogs, schemas, and tables you already have access to — if a
   table you expect isn't listed, it's a permissions question, not a bug.
4. The table detail opens.

<Admonition type="tip" title="Adding many tables at once">
Need to bring dozens of tables under monitoring? Use bulk-register to add many
tables in one step instead of opening them one at a time.
</Admonition>

## The table detail

A monitored table has four tabs:

- **About** — the table's schema, a preview of sample data, and an "ask a
  question" box for quick natural-language questions about the data.
- **Apply Rules** — where you turn rules into live checks. See
  [Assign rules](/docs/studio/monitoring/assign-rules).
- **Results** — the table's data-quality score, its trend over time, and a
  breakdown you can drill into down to the failing rows. See the
  [Results page](/docs/studio/running/results).
- **Profile** — a scan of the table's shape that can suggest candidate rules. See
  [Profiling](/docs/studio/monitoring/profiling).

## In this section

- **[Assign rules](/docs/studio/monitoring/assign-rules)** — attach rules from
  your registry to a table's columns and tune them per table.
- **[Set quality thresholds](/docs/studio/monitoring/quality-thresholds)** —
  require, say, 95% of rows to pass and improve quality gradually.
- **[Profiling](/docs/studio/monitoring/profiling)** — scan a table and let the
  data suggest candidate rules.
- **[Collections](/docs/studio/monitoring/collections)** — group related tables
  into a data product and see quality across all of them at once.
