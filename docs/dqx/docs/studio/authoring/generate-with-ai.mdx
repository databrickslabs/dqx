# Generate rules with AI

If you'd rather describe a rule in plain English than build it by hand, let the app draft it for you. This page is for stewards who know the expectation they want to enforce but don't want to pick a check and fill in its settings themselves.

## Describe what you want

Open the rule editor (**Rules Registry → New rule**). At the top of the editor you'll find a **Generate** box. Type a plain-English description of what the data should look like, and the app proposes a check with its settings already filled in.

  alt="The rule editor with the Generate box at the top, where you describe a rule in plain English"

Good descriptions are specific about the values you expect. For example:

- `order amount must be positive and under $1,000,000`
- `this column should only contain email addresses`
- `status must be one of placed, shipped, delivered, cancelled`
- `signup date can't be in the future`
- `customer id is required and never blank`

The app reads your description, picks a matching check from the [checks catalog](/docs/studio/authoring/checks-catalog), and fills in its arguments — the allowed values, the range, the pattern, and so on.

## Review, test, then save

A generated rule is a **starting point**, not a finished rule. Read what the app proposed and confirm it matches what you meant. Adjust the name, description, severity, or any argument as you like — it behaves exactly like a rule you built yourself.

Before you save, open the **Test** tab and run the rule against sample rows. Testing changes nothing and saves nothing; it just shows you how the rule behaves so you can catch a mistake early.

<Admonition type="caution" title="Always review AI output">
AI can misread a description or choose settings you didn't intend. Never save a generated rule without reading it and running the Test tab first.
</Admonition>

Once saved, a generated rule follows the same path as any other rule — it starts as a draft, goes through review and approval, and every change is recorded in the audit history.
