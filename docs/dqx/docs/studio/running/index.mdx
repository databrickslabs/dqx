# Run & review

Once a table is monitored, you **run** its checks to see how the data is doing,
then **review** the results and act on anything that's failing.

- **[Run checks & schedule](/docs/studio/running/run-and-schedule)** — run on
  demand with **Run now**, or put a table or collection on an hourly, daily,
  weekly, or custom schedule. See past runs in Runs History.
- **[Results page](/docs/studio/running/results)** — read the DQ score and
  trend, break results down by dimension, severity, rule, and column, and drill
  into the failing records.
- **[Ask Genie](/docs/studio/running/genie)** — ask questions about your results
  in plain English and get an answer with a chart.
