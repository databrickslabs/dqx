# DQX App - Feature Implementation Summary

## Overview

This document summarizes the features implemented for the DQX (Data Quality) App, a Databricks application for managing data quality rules through profiling, rule generation, approval workflows, and validation.

---

## Features Implemented

### 1. Data Profiler (`/profiler`)

**Purpose**: Profile tables to automatically generate data quality rules.

**Functionality**:
- Select catalog, schema, and table from dropdowns (populated dynamically)
- Creates a temporary view for profiling
- Uses `DQProfiler` to analyze data and `DQGenerator` to generate quality rules
- Displays profiling results (rows profiled, columns profiled, duration)
- Shows generated rules with checkboxes for selection
- Save selected rules to the rules table

**Key Components**:
- `ProfileTableSection` - Main profiling UI
- `ExistingProfilesList` - Shows previously run profiles for the selected table

### 2. Quality Rules Management (`/rules`)

**Purpose**: View, filter, approve, and export data quality rules.

**Functionality**:
- **Filtering**: Filter rules by:
  - Catalog
  - Schema
  - Table
  - Status (Draft, Pending Approval, Approved, Exported, Rejected)
- **Rule Display**: Each rule card shows:
  - Status badge with color coding
  - Source table FQN
  - Actual rule checks (check type, column, criticality, parameters)
  - Created by / date
  - Approval info (if approved)
- **Dry Run**: Test rules against sample data (10,000 rows) with:
  - Pass rate progress bar (color-coded: green вүҘ90%, yellow вүҘ70%, red <70%)
  - Passed/failed row counts
  - Results displayed inline in the rule card
- **View Details**: Full rule details in a dialog
- **Approval Workflow**:
  - Draft вҶ’ Submit for Approval вҶ’ Pending Approval
  - Approve or Reject pending rules
- **Export to Checks**: Export approved rules for use by DQX engine

### 3. Configuration Storage

**Purpose**: Persist app configuration in Delta tables instead of local files.

**Tables Created**:
| Table | Purpose |
|-------|---------|
| `dq_app_settings` | App configuration (target catalog, initialized status) |
| `dq_profiling_results` | Historical profiling results |
| `dq_quality_rules` | Quality rules with approval workflow |

## API Endpoints Added

### Profiling
- `POST /api/profile-table` - Profile a table and generate rules
- `GET /api/existing-profiles` - Get existing profiles for a table

### Rules
- `GET /api/rules` - List rules with filters (catalog, schema, table, status)
- `GET /api/rules/filter-options` - Get unique values for filter dropdowns
- `POST /api/rules` - Create a new rule
- `GET /api/rules/{rule_id}` - Get a specific rule
- `PUT /api/rules/{rule_id}` - Update a rule
- `DELETE /api/rules/{rule_id}` - Delete a rule
- `POST /api/rules/{rule_id}/submit` - Submit rule for approval
- `POST /api/rules/{rule_id}/approve` - Approve a rule
- `POST /api/rules/{rule_id}/reject` - Reject a rule
- `POST /api/rules/{rule_id}/dry-run` - Run validation on sample data
- `POST /api/rules/{rule_id}/export` - Export approved rule to checks
